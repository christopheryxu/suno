System Prompt
A foundational set of hidden instructions given to an LLM that defines its role, behavior, and output format.
User Prompt
The visible input a user provides to the LLM. It is the question, task, or instruction that drives the conversation.
1. Context: Give the model the background it needs — who you are, what you're working on, and why.
2. Task: State exactly what you want the model to do — be specific and unambiguous.
3. Format: Define how the output should look — bullets, table, code, length, tone.
4. Examples: Show a sample of what good output looks like so the model can pattern-match.
Task-Model Fit:
The right model depends on the type of task and its complexity. There is a fundamental tradeoff between cost / latency and output accuracy / complexity.

The Need to Manage Context:
LLMs have no built-in memory, every conversation starts from zero. Context and memory systems give agents the information they need to be useful: past conversations, team knowledge, and connected data. Managing these well is the difference between a generic chatbot and a reliable teammate.
Type of Memory
Short-term: Conversation history within a single session.
Long-term: Persistent knowledge stored across sessions (e.g., CLAUDE.md, vector stores).
External: Connected data sources — CRM, docs, Slack, meeting transcripts.
Managing Context
Context windows: How much information a model can process at once (varies by model).
Retrieval (RAG): Pull relevant information from large datasets on demand.
Summarization: Compress long conversations to fit within context limits.
What Are Skills?
Reusable instructions that teach an AI agent how to perform a specific task consistently.
Skills encode team knowledge — style guides, review checklists, data formats — into repeatable prompts.
Examples by Team
Legal: Contract review skill that flags non-standard clauses.
Growth: Campaign brief generator from a few bullet points.
Finance: Expense categorization skill aligned to internal taxonomy.
LLMs become much more powerful when connected to real data and tools. Tool use allow agents to take actions such as reading from databases, sending messages, updating records. This turns conversations into workflows.
Types of Tools
MCP Servers: Connect AI agents to internal tools — Slack, Google Drive, CRM, databases.
APIs: Expose internal services so agents can read/write data programmatically.
Webhooks: Trigger AI workflows from events in existing systems.
What is MCP?
Model Context Protocol: An open standard that lets AI agents connect to external tools and data sources through a unified interface.
How it works: MCP servers expose capabilities (read files, query databases, send messages) that any compatible AI agent can discover and use.
Why it matters: Build once, connect anywhere — no custom integrations per tool or per model.
Why Validate?
AI outputs can be confident and wrong — especially for legal, financial, and compliance tasks.
Validation catches hallucinations, formatting errors, and missed edge cases before they reach users.
How to Validate
Build evals: Create automated evaluation pipelines that score model outputs against defined criteria — accuracy, format, tone, and completeness.
Golden test set: Curate a fixed set of representative inputs with known-good outputs. Run every prompt or model change against this set to catch regressions.

Human Review
What it is: Use a separate LLM to score or grade the output of another model against a rubric.
Best for: High-volume tasks where manual review doesn't scale — content generation, summarization, classification.
How to set up: Define scoring criteria in a system prompt, feed output + expected result, collect pass/fail + reasoning.
What it is: Domain experts manually review AI outputs for correctness, nuance, and edge cases.
Best for: High-stakes tasks — legal review, financial analysis, compliance — where errors have real consequences.
How to set up: Build a review queue, define acceptance criteria, sample a percentage of outputs for spot-checking.
Input Guardrails
Prompt injection: Detect and block adversarial inputs designed to override system instructions.
Jailbreaking: Prevent attempts to bypass model safety filters through crafted prompts.
Privacy: Filter PII and sensitive data before it reaches external models.
Output Guardrails
Hallucinations: Validate outputs against source data to catch fabricated facts or citations.
Data leakage: Scan responses for accidental exposure of internal or confidential information.
Toxicity: Filter outputs for harmful, biased, or inappropriate content before delivery.
Tracking Effective Usage
Team Dashboards: Monitor which teams use AI most and where adoption lags.
Tool-Level Metrics: Token consumption, API calls, and active users per tool.
Token Efficiency: Measure time savings, cost savings, or revenue driven as a function of tokens spent. The goal is ROI per token.
Managing Costs
Model Routing: Use cheaper models for simple tasks, reserve expensive models for complex ones.
Spend Limits: Set monthly budgets per team and alert before overruns.
Open-Source Models: Use hosted open-source inferencing (or self-host) where possible to reduce per-query costs at scale.
