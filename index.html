System Prompt
A foundational set of hidden instructions given to an LLM that defines its role, behavior, and output format.
User Prompt
The visible input a user provides to the LLM — the question, task, or instruction that drives the conversation.
The right model depends on the type of task and its complexity. There is a fundamental tradeoff between cost / latency and output accuracy / complexity.

Type of Memory
Short-term: Conversation history within a single session.
Long-term: Persistent knowledge stored across sessions (e.g., CLAUDE.md, vector stores).
External: Connected data sources — CRM, docs, Slack, meeting transcripts.
Managing Context
Context windows: How much information a model can process at once (varies by model).
Retrieval (RAG): Pull relevant information from large datasets on demand.
Summarization: Compress long conversations to fit within context limits.
What Are Skills?
Reusable instructions that teach an AI agent how to perform a specific task consistently.
Skills encode team knowledge — style guides, review checklists, data formats — into repeatable prompts.
Examples at Suno
Legal: Contract review skill that flags non-standard clauses.
Growth: Campaign brief generator from a few bullet points.
Finance: Expense categorization skill aligned to internal taxonomy.
Tool Integrations
MCP Servers: Connect AI agents to internal tools — Slack, Google Drive, CRM, databases.
APIs: Expose internal services so agents can read/write data programmatically.
Webhooks: Trigger AI workflows from events in existing systems.
Evaluation Criteria
Security: Does the tool meet Suno's data handling and access policies?
Cost: Per-seat vs. usage-based — what's the ROI at our scale?
Risk: What happens if the tool goes down or the vendor changes terms?
Why Validate?
AI outputs can be confident and wrong — especially for legal, financial, and compliance tasks.
Validation catches hallucinations, formatting errors, and missed edge cases before they reach users.
How to Validate
Human-in-the-loop: Review and approve AI outputs before they go live.
Automated checks: Schema validation, keyword matching, confidence thresholds.
A/B testing: Compare AI-generated vs. manual outputs on real tasks.
Iterative Refinement
Prompt tuning: Adjust instructions based on failure patterns and edge cases.
Feedback loops: Collect user feedback to improve prompts and workflows weekly.
Version control: Track prompt and skill changes over time.
LLM Evals
Accuracy: Does the output match the expected result on a test set?
Cost/speed: Compare models on latency and token cost for each task.
Regression testing: Ensure new prompts don't break existing workflows.
Usage & Costs
Tracking: Monitor per-team and per-tool AI usage to understand adoption patterns.
Budgeting: Set per-seat and per-project spend limits; review monthly.
ROI: Tie AI spend to measurable outcomes — hours saved, tasks automated, errors reduced.
Safety Guardrails
Data policies: Define what data can and cannot be sent to external models.
Access controls: Role-based permissions for AI tools and connectors.
Guidelines: Clear, practical AI usage guidelines — what's encouraged, what requires review.
Tracking Usage
Per-team dashboards: Monitor which teams use AI most and where adoption lags.
Tool-level metrics: Token consumption, API calls, and active users per tool.
Adoption signals: Track repeat usage, not just sign-ups — are people actually using it?
Managing Costs
Model routing: Use cheaper models for simple tasks, reserve expensive models for complex ones.
Spend limits: Set monthly budgets per team and alert before overruns.
Open-source models: Self-host where possible to reduce per-query costs at scale.
